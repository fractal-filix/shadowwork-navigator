## 1.6.3.1 開発環境（破壊的・再作成）

開発環境では `recreate-d1.ps1` を使って、DBを再作成して `database/DDL.sql` を適用する。

```powershell
cd apps/api
pwsh -File scripts/recreate-d1.ps1 -DatabaseName filix_shadowwork_dev -Binding DB -DeleteExisting
```

補足:
- この手順は `wrangler.toml` の `database_id` を更新するため、実行後は差分を確認する。
- `-DeleteExisting` は破壊的なので、**開発環境のみ**で使用する。

実行後の確認:

```powershell
pnpm exec wrangler d1 execute filix_shadowwork_dev --command "SELECT name FROM sqlite_master WHERE type='table' ORDER BY name;"
```

---

## 1.6.3.2 staging/production（非破壊・差分適用）

staging/production では DB を再作成せず、差分SQLを作って適用する。

補足（リリース前に破壊的適用する場合）:
- `scripts/recreate-d1.ps1` に `-RemoteExecute` を付けると、`wrangler d1 execute` が `--remote` 付きで実行される。
- 例（staging 破壊的再作成）:

```powershell
cd apps/api
pwsh -File scripts/recreate-d1.ps1 -DatabaseName filix_shadowwork_stg -Binding DB -Environment staging -DeleteExisting -RemoteExecute
```

```powershell
cd apps/api
pwsh -File scripts/recreate-d1.ps1 -DatabaseName filix_shadowwork_prod -Binding DB -Environment production -DeleteExisting -RemoteExecute
```

### 手順A: 差分SQLを作成

- 例: `apps/api/database/migrations/20260306_add_xxx.sql`
- 破壊的変更（`DROP TABLE`, `DELETE FROM`, 全件更新など）は入れない。
- まず staging に適用し、検証後に production へ展開する。

### 手順B: staging へ適用

```powershell
cd apps/api
pnpm exec wrangler d1 execute filix_shadowwork_stg --env staging --file database/migrations/<migration_file>.sql
```

検証クエリ例:

```powershell
pnpm exec wrangler d1 execute filix_shadowwork_stg --env staging --command "PRAGMA table_info(messages);"
pnpm exec wrangler d1 execute filix_shadowwork_stg --env staging --command "SELECT name FROM sqlite_master WHERE type='index' ORDER BY name;"
```

### 手順C: production へ適用

```powershell
cd apps/api
pnpm exec wrangler d1 execute filix_shadowwork_prod --env production --file database/migrations/<migration_file>.sql
```

検証クエリ例:

```powershell
pnpm exec wrangler d1 execute filix_shadowwork_prod --env production --command "PRAGMA table_info(messages);"
pnpm exec wrangler d1 execute filix_shadowwork_prod --env production --command "SELECT name FROM sqlite_master WHERE type='index' ORDER BY name;"
```

## 運用ルール（このスプリント中）
