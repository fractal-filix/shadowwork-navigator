## 1.2.5.2 管理者用アンラップ（Decrypt）の許可主体を確定

### 結論（運用方針）
- `kms:Decrypt`（アンラップ）は **管理者専用の IAM Role にのみ付与**します。
- 具体的には以下の組み合わせを推奨します。
  - 管理操作用 Role: `arn:aws:iam::555569220922:role/shadownav-admins`（Role 名は運用に合わせて変更可）
  - Role の信頼関係（Trust Policy）は、明示的に許可する少数の IAM ユーザ（例: `shadownav-admin1`）のみが `sts:AssumeRole` できるように限定する（下記参照）
  - Role 側のポリシーに `kms:Decrypt` を付与する際、MFA の有無を条件として要求する（`Bool":{"aws:MultiFactorAuthPresent":"true"}`）など追加条件を推奨

### 理由
- `kms:Decrypt` は秘密鍵操作に相当するため最小限の主体に限定すべき
- Cloudflare Workers 等の API 実行主体に `Decrypt` を付与すると、平文 DEK が Workers 内に露出するリスクが大きく、運用監査が困難になる

### 推奨ポリシー（Role にアタッチする最小例）
- ファイル: `docs/sprints/sprint1/aws/iam/kms_decrypt_admin_policy.json`（既に `KMS_KEY_ARN` を実値に置換済み）
- トラストポリシー雛形: `docs/sprints/sprint1/aws/iam/kms_decrypt_admin_trust_policy.json`

### AWS CLI の一例（管理者 Role / Policy を作る）
以下は AWS コンソール / CLI で実行する例です。管理者ユーザを作らない方針のため、まずは Role と Policy を作成し、信頼主体・AssumeRole の権限は運用者の判断で与えてください。

1) ポリシー作成（JSON を作成済みの場合）

```bash
aws iam create-policy --policy-name shadownav-kms-decrypt-admin --policy-document file://docs/sprints/sprint1/aws/iam/kms_decrypt_admin_policy.json
```

2) Role 作成（Trust policy を利用）

```bash
aws iam create-role --role-name shadownav-admins --assume-role-policy-document file://docs/sprints/sprint1/aws/iam/kms_decrypt_admin_trust_policy.json
```

3) Role にポリシーをアタッチ

```bash
aws iam attach-role-policy --role-name shadownav-admins --policy-arn arn:aws:iam::555569220922:policy/shadownav-kms-decrypt-admin
```

4) （運用）AssumeRole を許可する主体を最小化する

 - 可能ならば個別の `arn:aws:iam::555569220922:user/<admin>` を `kms_decrypt_admin_trust_policy.json` に明示して置き換えてください。
 - あるいは、既存の管理者 Role を信頼する形でトラストを整備してください。

### Role / User の作成手順（概要）
1) IAM → Roles → Create role
   - Role name: `shadownav-admins`
   - Trusted entities: 指定する管理者 IAM User の ARN のみを Principal に設定（下記の Trust policy 参照）
2) Role に `kms_decrypt_admin_policy` をアタッチ（後述の JSON を利用）
3) 管理者ユーザ側は必要に応じて `sts:AssumeRole` で Role を引き受け、CloudTrail で監査可能にする

### Trust policy（例）
以下の Trust policy を Role に設定し、AssumeRole を許可する主体を限定します（`<AWS_ACCOUNT_ID>` とユーザ名を実値に置換）。

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": [
          "arn:aws:iam::<AWS_ACCOUNT_ID>:user/shadownav-admin1",
          "arn:aws:iam::<AWS_ACCOUNT_ID>:user/shadownav-admin2"
        ]
      },
      "Action": "sts:AssumeRole",
      "Condition": {}
    }
  ]
}
```

### 監査・運用上の注意
- CloudTrail で `kms:Decrypt` 呼び出しを必ず監査し、誰がいつアンラップしたかが追跡できることを確認する
- アンラップ操作には「操作理由」のメタを別途記録するワークフロー（例: 管理者操作ログの記録）を推奨
