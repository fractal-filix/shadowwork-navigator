# 2026.03.13 Runbook（7.1 Qdrant接続確認）

このドキュメントは `docs/sprints/sprint1/task.md` の **2026.03.13 / 7.1** に対応する手順書です。

対象タスク:
- 7.1 Qdrant: 環境（Qdrant Cloud等）を確定、APIキー/TLSで接続

---

## 目的

- API から Qdrant Cloud へ **HTTPS（TLS）+ APIキー** で接続できることを確認する。
- 後続タスク（7.2 以降）の前提として、接続確認の再現手順を固定化する。

---

## 事前条件

- 作業ディレクトリ: `apps/api`
- Qdrant Cloud のエンドポイント（`https://...`）を確定済み
- Qdrant API キーを発行済み
- （任意）対象 collection 名を確定済み

必要環境変数:
- `QDRANT_URL`
- `QDRANT_API_KEY`
- `QDRANT_COLLECTION`（任意）

---

## 実行手順

PowerShell の例:

```powershell
cd apps/api
$env:QDRANT_URL = "https://<cluster>.<region>.qdrant.cloud"
$env:QDRANT_API_KEY = "<qdrant_api_key>"
$env:QDRANT_COLLECTION = "<collection_name>" # 任意
pnpm run verify:qdrant
```

期待結果:
- `Qdrant connectivity check: OK`
- `- endpoint: https://...`
- `- tls: https`
- `- collections: <number>`
- `QDRANT_COLLECTION` を設定した場合は `found` が表示される

---

## 実装内容（この runbook で追加）

- `apps/api/scripts/verify-qdrant.mjs`
  - `QDRANT_URL` が `https://` であることを検証
  - `GET /collections` を `api-key` ヘッダー付きで実行
  - `QDRANT_COLLECTION` 指定時は存在チェックを実施
- `apps/api/package.json`
  - `verify:qdrant` スクリプトを追加

---

## 補足

- このチェックは接続性と基本設定の確認を目的とし、upsert/search 実装は 7.2 で行う。
- APIキーは Secrets として管理し、ログやドキュメントに実値を残さない。
