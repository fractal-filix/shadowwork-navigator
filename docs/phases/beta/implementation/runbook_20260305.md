# 2026.03.05 Runbook（アンラップ（Decrypt）許可主体）

このドキュメントは `task.md` の **2026.03.05 / 1.2.5.2** に対応する「アンラップ（Decrypt）の許可主体確定」手順書です。

用語:
- アンラップ = `kms:Decrypt`（ラップ済み DEK の復号）
- API 実装名: `POST /api/crypto/dek/unseal`（「unseal」という語を使用）

前提（β）:
- 人間管理者用の IAM ユーザーは作成しない（必須ではない）
- 既存の運用主体（例: `user/shadownav-workers`、AWS SSO の Role 等）を前提に最小権限で設計する

---

## 結論（Principal を分ける）

1) 公開鍵取得（GetPublicKey）用（弱い権限）
- Role: `arn:aws:iam::555569220922:role/shadownav-workers-kms-role`
- Attached policy: `shadownav-kms-get-public-key`

2) アンラップ（Decrypt）用（強い権限）

2-1) 人間管理者用（break-glass）
- Role: `arn:aws:iam::555569220922:role/shadownav-admins`
- Attached policy: `../aws/iam/kms_decrypt_admin_policy.json`（MFA 条件あり）
- Trust policy: `../aws/iam/kms_decrypt_admin_trust_policy.json`
  - 注意: `root` 信頼は最終的に AWS SSO の特定 Role 等へ絞る（TODO）

2-2) Workers（API）用（βでアプリから Decrypt する場合）
- Role: `arn:aws:iam::555569220922:role/shadownav-workers-kms-decrypt-role`
- Trust policy（最小）: `../aws/iam/kms_decrypt_workers_trust_policy.json`
  - Principal: `arn:aws:iam::555569220922:user/shadownav-workers` のみに限定
- Attached policy（MFA 条件なし）: `../aws/iam/kms_decrypt_workers_policy.json`

理由:
- `kms:Decrypt` は秘密鍵操作に相当するため最小限の主体に限定すべき。
- Workers は MFA を伴えないため、MFA 条件付き policy をそのまま流用しない。

---

## 手順（Workers から Decrypt する場合）

### 1) Workers 用 Decrypt policy を作成

```bash
aws iam create-policy --policy-name shadownav-kms-decrypt-api --policy-document file://docs/phases/beta/aws/iam/kms_decrypt_workers_policy.json
```

### 2) Workers 用 Decrypt role を作成

```bash
aws iam create-role --role-name shadownav-workers-kms-decrypt-role --assume-role-policy-document file://docs/phases/beta/aws/iam/kms_decrypt_workers_trust_policy.json
```

### 3) Role に policy をアタッチ

```bash
aws iam attach-role-policy --role-name shadownav-workers-kms-decrypt-role --policy-arn arn:aws:iam::555569220922:policy/shadownav-kms-decrypt-api
```

### 4) Workers の `ASSUME_ROLE_ARN` を更新

Cloudflare Workers 側の Secret `ASSUME_ROLE_ARN` を以下へ変更します。

- `arn:aws:iam::555569220922:role/shadownav-workers-kms-decrypt-role`

補足（ローカル検証向け）:

- `aws kms get-public-key` の出力を手でコピペして PEM を作ると、`KeySpec` や `EncryptionAlgorithms` 等の文字列が混入して `InvalidCiphertextException` の原因になります。
- 公開鍵は必ず `--query PublicKey --output text` の base64 から PEM を生成してください。

PowerShell 例（`public_from_kms.pem` を生成）:

```powershell
$keyArn = 'arn:aws:kms:ap-southeast-2:555569220922:key/ea3e43d1-bc62-47d5-af00-fc609843a46f'
$b64 = aws kms get-public-key --key-id $keyArn --region ap-southeast-2 --query PublicKey --output text
$lines = @(); for ($i=0; $i -lt $b64.Length; $i+=64) { $lines += $b64.Substring($i, [Math]::Min(64, $b64.Length-$i)) }
"-----BEGIN PUBLIC KEY-----`n$($lines -join "`n")`n-----END PUBLIC KEY-----`n" | Set-Content -NoNewline .\public_from_kms.pem
```
